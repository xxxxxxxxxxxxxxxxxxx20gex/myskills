# Domain Playbooks

Use domain playbooks to choose project-specific nouns, page structures, and visual metaphors.

## Healthcare AI / Chinese Medicine Diagnosis

Common pages:
- Data governance
- Knowledge graph
- Diagnosis assistance workflow
- Traceability
- Safety review
- Expert audit
- Clinical decision boundary

Frequent entities:
- Literature, guideline, case, prescription, medicine, symptom, syndrome, contraindication, review record, expert feedback.

Recommended visuals:
- Evidence chain, audit panel, knowledge graph, diagnosis workflow, safety review gate, expert feedback loop.

Avoid:
- Implying the system replaces doctors.
- Mystical or decorative Chinese medicine visuals.
- Presenting recommendations as final clinical diagnosis without a review boundary.

## AI Research / Thesis Defense

Common pages:
- Motivation
- Related work gap
- Method framework
- Module design
- Loss function
- Experiment
- Ablation
- Comparison
- Limitation

Recommended visuals:
- Paper-figure pipeline, module replacement diagram, comparison table, ablation matrix, metric dashboard.

Avoid:
- Product marketing style for method pages.
- Decorative AI icons instead of variables, inputs, outputs, and modules.
- Adding metrics that are not provided.

## Enterprise Software / SaaS Platform

Common pages:
- Product overview
- User workflow
- Capability map
- System architecture
- Operations dashboard
- Integration architecture
- Permission and audit

Recommended visuals:
- Role-based workflow, capability grid, dashboard mock, API integration map, permission matrix.

Avoid:
- Unstructured feature lists.
- Generic "platform empowers business" copy.
- Fake dashboard values.

## Industrial Systems / Edge Intelligence

Common pages:
- Device-cloud architecture
- Data acquisition
- Edge inference
- Scheduling and control
- Fault monitoring
- Digital twin

Recommended visuals:
- Device-edge-cloud layers, telemetry pipeline, state machine, topology, risk-control loop.

Avoid:
- Consumer app visual style.
- Ignoring physical constraints such as latency, bandwidth, environment, and maintenance.

## Government / Public Sector Projects

Common pages:
- Background and policy alignment
- Construction goals
- Overall architecture
- Implementation path
- Governance mechanism
- Expected outcomes

Recommended visuals:
- Goal-system-measure structure, phased roadmap, governance matrix, outcome strip.

Avoid:
- Casual tone.
- Overly trendy visuals.
- Claims without policy, scenario, or outcome support.

## Data Governance / Compliance

Common pages:
- Data source inventory
- Data lifecycle
- Permission model
- Data lineage
- Quality control
- Audit and review

Recommended visuals:
- Lifecycle loop, lineage chain, permission matrix, quality dashboard, audit trail.

Avoid:
- Treating governance as a decorative shield icon.
- Omitting ownership, process, record, and review boundaries.
