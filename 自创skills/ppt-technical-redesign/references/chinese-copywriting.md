# Chinese Technical Copywriting

Use this file when rewriting Chinese PPT titles, subtitles, module names, bullets, and technical descriptions.

## Titles

Good titles use a technical noun plus a mechanism, path, system, or result.

Good:
- 数据可追溯机制：构建可回放、可审计、可复核的诊疗证据链
- 多源知识融合：打通文献、病例与方剂知识的结构化映射路径
- 安全审查闭环：从检索召回到处方建议的风险控制链路
- 轻量化部署方案：面向边缘设备的低延迟推理链路

Weak:
- 打造智能医疗新体验
- 让诊疗更智能
- 全面提升系统能力
- 构建先进平台

## Subtitles

Use the pattern: scenario + method + output boundary.

Examples:
- 面向严肃诊疗辅助场景，系统不仅给出答案，还记录依据来源、处理路径、调用过程与复核边界。
- 针对多源异构数据，平台通过统一解析、实体对齐和图谱索引形成可检索的知识底座。
- 在不改变主干模型的前提下，引入轻量适配模块提升跨域泛化能力。

Avoid:
- Repeating the title.
- Empty value claims.
- Long sentences with more than two clauses.

## Module Names

Prefer 4-8 Chinese characters. Use stable technical nouns.

Examples:
- 来源登记
- 片段解析
- 图谱映射
- 结论回放
- 安全复核
- 指标对齐
- 反馈闭环
- 权限校验
- 状态监控
- 异常定位

## Bullets

Use noun phrases. Keep bullets short enough for visual layout.

Good:
- 文档级来源标识
- 病例脱敏版本
- 原文片段回链
- 关系类型与置信依据
- 处方安全复核

Weak:
- 这里会记录每个文档的来源，方便后续查看
- 可以帮助用户更好地理解系统
- 系统会对各种数据进行处理

## Tone

- Use formal technical Chinese for research, government, and enterprise decks.
- Use compact product Chinese for SaaS or customer-facing pages.
- Avoid exaggerated promises such as "完全解决", "绝对安全", "全面替代", or "革命性突破" unless the source material proves it.
