# Layout Patterns

Use these patterns to convert plain slide content into a technical page.

## Blue-White Technical Architecture Page

Best for: AI systems, smart transportation, industrial software, project proposal decks.

Structure:
- Top: title plus one-line takeaway.
- Body: two or three large numbered modules.
- Inside each module: iconized subsystem, short label, 1-2 evidence captions, arrows showing actual flow.
- Bottom: project value strip with 3-4 measurable outcomes or differentiators.

Visual cues:
- Light blue-white gradient background.
- Thin technical border, grid/circuit accents, compact labels.
- Deep blue headings, cyan connectors, yellow/green highlight for final result.

## Data Pipeline / Model Pipeline

Best for: perception, OCR, multimodal fusion, model inference, backend workflow.

Structure:
- Left: input sources and constraints.
- Middle: processing stages with numbered nodes.
- Right: outputs, decisions, user-facing value.
- Add a feedback loop only if the system truly learns, caches, retries, or monitors.

Details to add:
- Input format, model name, key algorithm, storage/cache, latency/reliability note, output artifact.

## Problem-Solution-Proof

Best for: project overview, background, value proposition.

Structure:
- Left: 3 pain points with consequence labels.
- Center: proposed technical solution with architecture sketch.
- Right: proof points, metrics, or expected outcomes.

Do not make the pain points longer than the solution.

## Capability Grid

Best for: feature lists that currently look like bullets.

Structure:
- Group capabilities into 3-5 clusters.
- Each cluster gets icon, technical label, concrete function, and project-specific evidence.
- Use tags for real-time, low-cost, robust, scalable, edge/cloud, multimodal, explainable.

## Comparison Matrix

Best for: innovation points, competitive comparison, method selection.

Structure:
- Rows: evaluation dimensions that matter to the project.
- Columns: baseline methods versus proposed method.
- Highlight only the winning cells; avoid coloring the whole table.
- Add a short conclusion banner under the table.

## Technical Route Page

Best for: implementation plan, methodology, system design.

Structure:
- Use 4-6 stages in a horizontal route.
- Each stage shows input, method, output.
- Under the route, add risk/control or validation checkpoints.

## Density Checklist

Before finalizing, ensure the slide has:
- One clear page thesis.
- A diagram or structured layout as the primary object.
- Project-specific nouns, not generic labels.
- Enough labels for someone to understand the flow without narration.
- No paragraph block that should be a diagram.
- No decorative element that competes with content.
- Text that fits after rendering.
