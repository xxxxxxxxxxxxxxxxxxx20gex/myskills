# Page Pattern Selection

Use this file when deciding how a slide should be structured.

## Background / Policy Alignment

Use:

- policy trend timeline
- external-driver chain
- goal-system-measure matrix
- governance loop

Good for explaining why the project is necessary.

## Pain Point

Use:

- source-to-failure funnel
- risk chain
- before/after contrast
- issue map with consequence labels

Keep pain shorter than the proposed solution.

## Architecture

Use:

- layered architecture
- data pipeline
- hub-and-spoke map
- module dependency map

Show inputs, processing modules, storage, model/agent layer, outputs, and control boundaries.

## Workflow / Algorithm

Use:

- numbered route
- state machine
- swimlane flow
- input-method-output stages
- feedback loop only when there is real retry, review, learning, or monitoring

## Governance / Safety

Use:

- evidence chain
- audit trail
- safety review gates
- permission/review matrix
- quality-control loop

Make ownership, record, review, and boundary visible.

## Innovation / Value

Use:

- comparison matrix
- four-quadrant innovation map
- proof strip
- old/new capability contrast

Avoid generic "platform empowerment" phrasing; use concrete functions and outcomes.
